How to use it

Start with today's card. Complete only the highest-priority tasks first, write notes where the work happened, and treat every orange decision badge as something that needs a real answer.
Progress is saved locally in your browser. Use the Excel version as the durable project record; use this page as a lightweight daily cockpit.

Build in Public — later, not now

Use this once the design is validated and the build has started safely. Suggested trigger: Design Freeze passed + rolling chassis completes a safe flat-ground test. Until then, collect private photos, test notes, and lessons learned.

  • Public update structure: what changed, what was tested, what failed, what happens next.
  • Do not publish unsafe test locations, unapproved sponsor claims, or anything that conflicts with event rules.
  • Make the story useful: decisions, trade-offs, tests, fixes. Less hype fog, more build signal.
